# Lab 03 - Building ERDs

This project was completed for **Lab 03** of the **CS355 / CE373 Database Systems** course at **Habib University**.

The lab focuses on building an **Entity Relationship Diagram (ERD)** for a given real-world scenario using **DB Designer**. In this lab, the task was to design an ERD for a **Cricket Tournament Management System** by identifying the correct entities, attributes, primary keys, foreign keys, relationships, and cardinalities. 

---

## Objective

The objective of this lab is to develop data modeling skills by learning how to design the data model of a business system. The main focus is on identifying:

- entities
- attributes
- relationships
- cardinalities
- primary keys
- foreign keys

The lab manual specifically states that the ERD should include entities, attributes with data types and null/not null constraints, primary keys, foreign keys, and relationships such as 1-1 and 1-M.

---

## Scenario

The assigned scenario for this lab was **Scenario 1 - Cricket Tournament**.

The system had to model a cricket tournament in Pakistan where:

- 8 different countries are participating
- each country nominates 12 players
- stadiums, cities, and capacities are recorded
- a panel of umpires from different countries is maintained
- each match has two selected umpires
- matches include date, time, participating teams, stadium, city, winning team, and man of the match
- the system stores total runs scored and wickets taken by each player in every match
